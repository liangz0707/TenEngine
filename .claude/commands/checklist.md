# Create validation checklist

## Purpose

Generate checklists for validation before/during implementation.

## Checklist Categories

### UX/UI Checklist
- User flow validation
- Accessibility requirements
- Responsive design verification

### Security Checklist
- Input validation
- Authentication/authorization
- Data protection

### Performance Checklist
- Resource usage limits
- Response time requirements
- Scalability verification

### Testing Checklist
- Unit test coverage
- Integration test scenarios
- Edge case coverage

### Code Quality Checklist
- Code style compliance
- Documentation completeness
- Error handling coverage

## Format

```markdown
# <Category> Checklist

## <Section>
- [ ] Item 1
- [ ] Item 2
- [X] Completed item

## Status
- Total: N
- Completed: M
- Incomplete: K
```

## Usage

Run before `/implement` to validate prerequisites.
Checklists must pass (or user approve) before implementation proceeds.
